## Architecture

Single-page app with view switching. No router; views are sections toggled by `switchView(name)`:

- `dashboard` — KPIs + 4 charts
- `expenses` — table with filters, CRUD via modal
- `income` — table with filters, CRUD via modal
- `budget` — per-category monthly limits with progress bars
- `import-export` — Excel import/export, Drive sync settings, danger zone

### State (single global `state` object)

```js
{
  expenses: [{ id, date, category, description, amount, createdAt }],
  income:   [{ id, date, source,   description, amount, createdAt }],
  budgets:  { [category]: number }, // monthly limits
  theme:    "light" | "dark",
  filters:        { from, to, category, search },
  incomeFilters:  { from, to, source, search },
  view: "dashboard"
}
```

Persisted to `localStorage` under key `expense_tracker_v1`. Drive sync mirrors this same shape as JSON.

### Constants

- `CATEGORIES = ["Food","Transport","Housing","Entertainment","Health","Shopping","Education","Other"]`
- `INCOME_SOURCES = ["Salary","Freelance","Investment","Business","Gift","Other"]`
- `CATEGORY_COLORS = { Food: "#ef4444", ... }` — used in charts and CSS badges (`.cat-Food`, etc.).

To add a category or income source: update the constant **and** add a matching `.cat-X` / `.inc-X` CSS class with a color.

### Key functions (search by name in `index.html`)

| Function | What it does |
|---|---|
| `load()` / `save()` | localStorage read/write. `save()` is wrapped to also trigger Drive sync. |
| `renderDashboard()` | KPIs + calls 4 chart drawers. |
| `drawCategoryChart`, `drawMonthlyChart`, `drawIncomeVsExpenseChart`, `drawDailyChart` | Each chart, isolated. |
| `renderExpenses()` / `renderIncome()` / `renderBudget()` | Table/grid render for each view. |
| `openExpenseModal()` / `openIncomeModal()` / `openBudgetModal()` | Modal open/edit. |
| `submitExpense()` / `submitIncome()` / `submitBudget()` | Form handlers. |
| `exportXlsx()` / `importXlsx(file)` | Excel pipeline. Export creates 4 sheets: Expenses, Monthly Summary, Income, Cash Flow. |
| `driveConnect()` / `pullFromDrive()` / `pushToDrive()` / `findOrCreateFile()` | Google Drive sync. |
| `scheduleDriveSync()` | Debounced (1.5s) auto-push, called from wrapped `save()`. |

### Drive sync flow

1. User pastes OAuth Client ID in Import/Export view → saved to `localStorage["drive_client_id"]`.
2. Click **Connect** → loads GAPI + GIS → `requestAccessToken` → on success, calls `findOrCreateFile()`.
3. `findOrCreateFile()` searches Drive for `expense-tracker-data.json`; if missing, creates it (mimeType `application/json`, multipart upload), then seeds with current local state.
4. Connection auto-pulls once. After that, every `save()` schedules a debounced push.
5. **Manual pull** button exists for cross-device refresh (no auto-pull yet — see "Common edits" below).
6. Scope: `drive.file` — app only sees files it created.

## Common edits — where to start

| Change | Edit |
|---|---|
| Add/remove an expense category | `CATEGORIES` constant + matching `.cat-Foo` CSS rule + (optional) color in `CATEGORY_COLORS`. |
| Add/remove an income source | `INCOME_SOURCES` constant + matching `.inc-Foo` CSS rule. |
| Add a new chart | Add `<canvas>` in dashboard `chart-grid`, write `drawXChart()`, call from `renderDashboard()`. |
| Change KPI cards | Edit `.kpi-grid` HTML in dashboard view + corresponding values in `renderDashboard()`. |
| Add a new sheet to Excel export | Add to `exportXlsx()` after existing `XLSX.utils.book_append_sheet` calls. |
| Auto-pull on app open | In `driveConnect()` callback or `init()`, after token is restored, call `pullFromDrive()` periodically (e.g. `setInterval`) or on `visibilitychange`. |
| Add currency switching | Replace the `fmt` Intl.NumberFormat with one driven by a setting; persist in `state`. |
| Service worker cache bust | Bump `CACHE = "expense-tracker-vN"` in `sw.js`. Required when CDN URLs or filenames change. |

## Known limitations / decisions

- **Last-write-wins** Drive sync. No conflict resolution — fine for single user, multiple devices used sequentially.
- **No auto-pull** on app focus; user must hit "Pull from Drive" when switching devices. Could be added.
- **Test users only** OAuth — until OAuth consent screen is published. Each new user must be added in Google Cloud Console.
- **Service worker** caches Chart.js + SheetJS aggressively. Updating the CDN versions requires a cache version bump.
- Dates use ISO `YYYY-MM-DD` strings everywhere; `monthKey(d)` slices `0..7`. Don't introduce `Date` objects in state.
- Modals use a single overlay-click + Escape-key pattern; if you add a modal, register it in both close listeners.

## Regenerating icons

```bash
cd /Users/nassartor/Desktop/expense-tracker
python3 -c "
from PIL import Image, ImageDraw, ImageFont
def make(size, path):
    img = Image.new('RGB', (size,size), (99,102,241))
    grad = Image.new('RGBA',(size,size),(0,0,0,0))
    dg = ImageDraw.Draw(grad)
    for i in range(size):
        dg.line([(0,i),(size,i)], fill=(167,139,250,int(120*i/size)))
    img = Image.alpha_composite(img.convert('RGBA'),grad).convert('RGB')
    d = ImageDraw.Draw(img)
    f = ImageFont.truetype('/System/Library/Fonts/Supplemental/Arial Bold.ttf', int(size*0.55))
    bb = d.textbbox((0,0),'\$',font=f)
    d.text(((size-(bb[2]-bb[0]))/2-bb[0],(size-(bb[3]-bb[1]))/2-bb[1]-int(size*0.04)),'\$',font=f,fill=(255,255,255))
    m = Image.new('L',(size,size),0)
    ImageDraw.Draw(m).rounded_rectangle((0,0,size,size), radius=int(size*0.22), fill=255)
    out = Image.new('RGBA',(size,size),(0,0,0,0)); out.paste(img,(0,0),m); out.save(path,'PNG')
make(180,'apple-touch-icon.png'); make(192,'icon-192.png'); make(512,'icon-512.png')
"
```
